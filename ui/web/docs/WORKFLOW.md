# 使用流程

## 日常推进流程
1. 从 [INDEX.md](./INDEX.md) 进入对应阶段。
2. 阅读阶段 `README.md`，确认目标、边界、输入输出。
3. 执行 `TODO.md` 中的条目并逐项打勾。
4. 每项任务完成时补充证据链接。
5. 阶段末执行 `ACCEPTANCE.md`，全部通过后再标记阶段 `Done`。
6. 新决策或方案变更写入 `DECISIONS.md`。

## 任务状态规则
- `Not Started`: 尚未开始。
- `In Progress`: 正在执行。
- `Blocked`: 因外部依赖或未决决策阻塞。
- `Done`: 已完成并具备证据。

## 证据规则
- 最低要求：每个 `Done` 任务必须有证据链接。
- 可用证据：PR、commit、测试命令与结果、UI截图、日志片段。
- 禁止无证据勾选任务。

## 变更治理
- 需求新增或边界变化，先更新 Phase 01/02 文档，再进入实现阶段。
- 若实现中发现设计冲突，先回写 `DECISIONS.md`，再继续编码。
