# 06 发布迁移与风险

## 发布阶段

1. Phase A：引入新架构与兼容桥接（功能保持）。
2. Phase B：逐步将现网配置迁移到 `[model.models]`。
3. Phase C：统计桥接命中率并评估移除 legacy 时间。

## 灰度策略

1. 先在开发环境启用新 schema。
2. 再在测试环境启用阶段路由 + fallback。
3. 生产先单阶段使用，再逐步拆分 planner/final_writer 模型。

## 回滚策略

1. 配置回滚：恢复 legacy `[model]`。
2. 代码回滚：回退到单客户端工厂版本。
3. 运行时回滚：临时关闭路由 fallback（fallback 为空）。

## 主要风险与缓解

1. 配置复杂度提高：提供模板与启动校验错误提示。
2. fallback 导致响应变慢：限制最大 fallback 层数。
3. 协议差异导致输出不一致：契约测试覆盖关键字段。
