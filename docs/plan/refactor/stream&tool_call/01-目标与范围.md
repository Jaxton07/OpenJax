# 01 目标与范围

## 背景
当前默认路径以 PlannerOnly 为主，无法稳定提供真实模型级流式体验；工具调用仍为单决策串行模式，不满足高效并行任务需求。

## 重构目标
- 将对话主链路升级为真实流式优先。
- 支持单轮内批量 `tool_calls` 与依赖调度并发执行。
- 建立结构化 `tool_results` 回注，提升模型上下文质量。
- 在 gateway/web 侧保证事件完整性、可恢复性与流畅体验。

## 非目标
- 本期不引入全新产品目录或独立 v2 工程。
- 本期不改变业务权限模型，仅在现有审批框架内升级批量语义。

## 范围边界
- Core：回合状态机、决策 schema、工具调度、回注机制。
- Gateway：事件封装、缺口恢复、重放策略。
- Web：状态机、批处理渲染、工具批次展示。
- Protocol：事件契约与请求响应模型版本升级。

## 成功标准
- 首 token 延迟显著下降且稳定。
- 高频输出下无静默丢片与错序。
- 并发工具调用可追踪、可取消、可重试。
- 端到端测试覆盖流式、并发、恢复三类核心路径。
