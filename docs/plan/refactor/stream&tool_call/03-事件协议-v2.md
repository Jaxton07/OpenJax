# 03 事件协议 v2

## 设计目标
- 明确区分真实流式与降级流式来源。
- 提供可恢复的事件序列语义，避免静默丢片。
- 支持批量工具调用生命周期展示。

## 统一 Envelope
- `request_id`
- `session_id`
- `turn_id`
- `event_seq`
- `turn_seq`
- `timestamp`
- `type`
- `stream_source` (`model_live` | `synthetic` | `replay`)
- `payload`

## 事件类型
- `response_started`
- `response_text_delta`
- `tool_calls_proposed`
- `tool_call_started`
- `tool_call_completed`
- `tool_batch_completed`
- `response_resumed`
- `response_completed`
- `response_error`
- `session_shutdown`

## 序列一致性与恢复
- 客户端必须校验 `event_seq` 连续性。
- 发现缺口后，使用 `after_event_seq` 请求重放。
- 服务端必须支持基于 `Last-Event-ID` 的恢复续流。
- 当窗口不足以补齐时，返回明确恢复失败语义并触发重连。

## 向后兼容
- v1/v2 在灰度期并行。
- 通过握手参数协商事件版本。
- 默认新客户端走 v2，旧客户端保持 v1。
