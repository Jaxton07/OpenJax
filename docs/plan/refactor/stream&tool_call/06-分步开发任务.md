# 06 分步开发任务

## Phase 0 文档与边界确认
- 产出完整文档包与术语统一。
- 确认代码组织策略与新增文件边界。
- 冻结 v2 契约草案进入实施。

## Phase 1 协议与数据模型
- 升级 protocol 事件与 envelope 字段。
- 升级模型决策 schema（支持 `tool_calls[]`）。
- 升级回注 schema（支持 `tool_results[]`）。
- 提供 v1/v2 协商与兼容层。

## Phase 2 Core 引擎
- 实现单阶段 Streaming Turn Engine。
- 接入真实流式主路径与受控降级路径。
- 实现批量工具调度器与依赖执行。
- 打通审批、取消、超时与错误恢复。

## Phase 3 Gateway 传输
- 发布 v2 事件并维持严格序列。
- 实现缺口检测、重放与重连策略。
- 补齐 shutdown/error 终态语义。

## Phase 4 Web 体验
- 重构会话状态机适配批量工具生命周期。
- 实现 delta 批处理渲染与草稿固化分层。
- 优化滚动跟随与长会话性能。

## Phase 5 验证与灰度
- 完成 core/gateway/web 测试矩阵。
- 执行灰度并评估关键指标。
- 收敛问题并推进默认路径切换。
