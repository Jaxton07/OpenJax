# 02 单阶段执行引擎架构

## 架构原则
- 单阶段流式回合：同一轮内同时处理文本输出与工具控制。
- 可中断与可恢复：工具批次执行期间暂停文本流，结果回注后续流。
- 事件优先：所有关键状态变化都映射为可消费事件。

## 目标状态机
- `turn_started`
- `streaming_text`
- `tool_batch_planned`
- `tool_batch_running`
- `tool_batch_completed`
- `streaming_resumed`
- `turn_completed`
- `turn_error`

## 关键组件
- `TurnEngine`：统一回合驱动与状态迁移。
- `DecisionParserV2`：解析批量 `tool_calls` 与文本增量控制指令。
- `ToolBatchScheduler`：依赖解析、并发执行、聚合结果。
- `ResultInjector`：将 `tool_results` 结构化回注模型继续生成。

## 关键流程
1. 启动回合并开启模型流式输出。
2. 解析输出片段：文本直接下发；工具计划进入批次调度。
3. 执行工具批次并持续上报每个调用生命周期事件。
4. 聚合批次结果并结构化回注模型，恢复文本生成。
5. 输出最终完成事件并固化会话状态。

## 兼容策略
- 保留旧路径开关，仅作为灰度兜底。
- 新路径默认开启后，旧路径进入受控退场。
