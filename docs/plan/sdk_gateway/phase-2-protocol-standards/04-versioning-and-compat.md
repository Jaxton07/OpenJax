# 04 Versioning and Compatibility

## 版本策略

- 路径版本：`/api/v1`。
- v1 内新增字段必须保持向后兼容（仅追加可选字段）。
- 删除/重命名字段或改变语义视为破坏性变更，进入 `v2`。

## 兼容规则

- 客户端必须忽略未知字段。
- 网关不得移除已发布字段，除非升级主版本。
- 事件 `type` 新增需文档登记，已有 `type` 语义不可漂移。

## 变更流程

1. 先更新 phase-2 契约文档。
2. 在 `../DECISIONS.md` 记录变更原因、影响、迁移策略。
3. 再更新 phase-3/4/5 的引用文档。

## 弃用策略

- 被弃用字段在一个主版本周期内保留。
- 弃用字段需在文档标注 `deprecated` 与替代字段。
