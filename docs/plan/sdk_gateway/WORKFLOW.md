# SDK Gateway 文档工作流

本文件定义文档读取顺序、更新纪律和上下文控制规则。

## 1. 标准读取顺序（必须遵守）

1. 根 [README.md](./README.md)
2. 根 [TODO.md](./TODO.md)
3. 当前阶段 `README.md`
4. 当前阶段 `TODO.md`
5. 当前任务文档

## 2. 上下文控制规则（防膨胀）

- 默认禁止读取非当前阶段文档。
- 仅当当前文档存在“前置依赖链接”时，允许跨阶段读取。
- 已完成阶段在根 README 只保留三行摘要，不展开过程历史。
- 非当前任务相关内容不进入当次上下文。

## 3. 文档更新纪律

- 先更新文档，再执行实现（除紧急修复外）。
- 任务状态变更必须同步更新：
  - 根 `TODO.md`（阶段级状态）
  - 当前阶段 `TODO.md`（执行级状态）
- 方案变化先记 [DECISIONS.md](./DECISIONS.md)，再改各阶段文档。
- 新增文档必须挂到当前阶段 `INDEX.md`。

## 4. 阶段迁移规则

- 进入新阶段前，检查前一阶段 `Go/No-Go` 与 `DoD` 是否满足。
- 阶段切换时，只做两处全局更新：
  - 根 `README.md` 当前状态面板
  - 根 `TODO.md` 阶段状态

## 5. 最小执行闭环

1. 读根 README 和根 TODO 确认当前阶段。
2. 进入当前阶段 README 和 TODO 确认任务与验收。
3. 只阅读当前任务文档完成工作。
4. 回写阶段 TODO 与根 TODO 状态。
5. 如有策略变更，补充 DECISIONS。
